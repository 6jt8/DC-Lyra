# Contributing to Lyra-MC

Thanks for your interest in contributing! This document covers the basics to get you started.

## How to Help

- **Bug Reports**: Open an issue with a clear description, logs, and reproduction steps.
- **Feature Requests**: Open an issue describing the feature and its use case.
- **Pull Requests**: Small, focused changes are easiest to review. Large features should be discussed in an issue first.

## Development Setup

```bash
git clone https://github.com/6jt8/Lyra-MC.git
cd Lyra-MC
bun install
bun run build
bun start
```

## Rules

- Follow the existing code style.
- Do not commit secrets, keys, or `.env` files.
- Test your changes before submitting.
- Be respectful in issues and discussions.

## Questions

Open an issue with the label `question` if you need help.
